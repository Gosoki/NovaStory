# 数据字典与可计算指标(现状盘点)

> 建立 2026-07-02。回答"现有数据能算出什么":四张表 + events 事件流的完整能力清单,与缺口(对应 paper/8 §📋 LOG 计划)。
> 以代码为准:`core/db.py`(schema)、`core/state.py`(log_event/round_durations)、`views/_trial.py`(落库)、`views/_streaming.py`(LLM 事件)。

---

## 0. 事件流总览(events 表)

每行 = participant_id + round_idx + **ts(毫秒,7-02 深夜起)** + type + payload_json + **seq_in_round(轮内序号)** + **attempt(会话段 id,轮每次重开换新)** + **trial_id(提交后回填到中签 attempt)**。7-02 之前的旧行:ts 秒级、后三列 NULL。

一轮的完整时间线:

```
round_start
→ intent_submit {chars}
→ [E] guidance_shown {round, source, n_questions, fallback}
→ [E] guidance_answer_saved {round, q} ×每次翻页(7-02 起;相邻两条的毫秒差≈每问耗时)
→ llm_start {group} → llm_done {group, elapsed, usage:{prompt,completion,total_tokens}} / llm_error {group, elapsed[, kind]}
→ [E] guidance_answer ×每问 + guidance_submit {round}(两者都在终稿/修改稿生成**成功后补记**,落库顺序在 llm_done 之后)
→ script_shown {v}
→ hand_edit_saved {v, chars_delta} / revision_request {chars} / continue_guidance_click
→ (循环…)
→ trial_submit(→ 回填 trial_id:本 attempt **截至此刻**的事件;其后的 questionnaire_submit、最后一轮的 final_survey_submit 仍为 NULL)
→ questionnaire_submit
```
另有:`session_resumed`(刷新续接)、`final_survey_submit`、`dev_switch_condition`(研究员)。
`group` 取值区分每次调用类型:`C-final` / `D-final` / `D-revise` / `E-guidance-r1` / `E-guidance-fu` / `E-final` / `E-revise`。
**重做轮切段**:同 (participant, round) 下按 `attempt` 分组;进了论文的那次 = 组内含 `trial_id` 非空行的组。注意:不要用 `WHERE trial_id IS NOT NULL` 过滤事件重建时间线(会丢 questionnaire_submit 等提交后事件)——先按 trial_id 定位中签 attempt,再按 `(participant_id, round_idx, attempt)` 取全组。

## 1. 四个关键需求的可计算性(2026-07-02 用户提问)

### ① 每次提交了什么(内容级) — ✅ 完成的轮全文可得
| 提交物 | 存哪 | 粒度 |
|---|---|---|
| 创意原文 | `trials.intent_statement` | 全文 |
| **AI 生成的每一版脚本** | `trials.script_versions` 中 `author="ai"` 的版本 | 全文 |
| **用户手改后的每一版脚本** | 同上 `author="user_edit"` | 全文(快照硬规则:提交/任何 AI 动作前自动落版) |
| 最终提交稿 | `trials.final_output` | 全文 |
| D 每次反馈 | `trials.revision_requests` `[{round, text}]` | 原文+顺序 |
| E 每问答案 | `trials.guidance_json`(问题/选项/chosen/is_custom/ai_decided/fallback) | 全文 |
| 每轮问卷 | `questionnaires` 各列 | 逐题 |
| 总问卷 | `participants.final_survey_json` | 逐题 |
| 背景问卷 | `participants.demographics_json / screening_json` | 逐题(index) |

⚠️ 唯一缺口:**中途退出、没走到 trial_submit 的轮**,以上内容不落库(trials 行在提交时才写;events 只有动作类型+字数,无内容)——即 paper/8 LOG1 的动机。

### ② 点击生成按钮 → AI 生成完的时间 — ✅ 精确可得(浮点秒,不受 ts 秒级限制)
- 每次调用:`llm_done.payload.elapsed`(`time.time()` 差,2 位小数);失败调用 `llm_error.payload.elapsed` 同样有。
- "点击"时刻 ≈ `llm_start`(与按钮点击同一次 rerun,间隔毫秒级;D 的 `revision_request` 事件是成功后才记的,点击时刻以 `llm_start` 为准)。
- 轮级合计:`trials.t_llm_wait`(含失败等待,7-02 起)。
- 口径注意:`elapsed` 含首 token 前自动重试的总耗时 = **用户实际等待**口径;"模型纯生成/首 token 延迟"口径在运维日志 `data/llm.log`(start/first_token/done 行,按 ts+user 对齐)。
- 7-02 起每次调用还带 `usage`(token 成本,取不到为 null)。

### ③ 脚本显示 → 用户下次点击按钮的时间 — ✅ 毫秒级可得(7-02 起;旧行秒级);有无修改三重可判
- 窗口 = `script_shown{v}` → 下一个动作事件:C 是 `trial_submit`;D 是 `llm_start(D-revise)`(=点击发送)或 `trial_submit`;E 是 `continue_guidance_click` 或 `trial_submit`。`hand_edit_saved` 与触发它的点击同刻落账(带 `chars_delta`)。
- **有无修改**:① `n_hand_edits`>0;② `hand_edit_chars`(difflib 变更字符量);③ `script_versions` 里 user_edit 版与前一 ai 版逐字 diff(全文都在,任意粒度可算);终稿 vs 最后 AI 版亦可直接比。
- 轮级净时长:`t_postgen`(首版稿→提交,已扣所有 AI 等待)。
- ⚠️ 限制:窗口内"阅读 vs 打字"分不开(编辑器无键入埋点,快照只在点击时落)。~~事件 ts 秒级~~ → 7-02 起毫秒。

### ④ 问卷显示 → 问卷结束的时间 — ✅ 毫秒级可得(7-02 起;旧行秒级)
- 每轮:`trial_submit`(提交即切问卷,下一帧渲染,≈显示时刻)→ `questionnaire_submit`,ts 差即问卷时长;`questionnaires.created_at` 同秒佐证。
- 总问卷:最后一轮 `questionnaire_submit` → `final_survey_submit`。
- 注意:`t_total`(round_start→trial_submit)**不含**问卷时间,问卷时长须单独从 events 算。

## 2. 完整可计算清单

### 被试级
- 人口学/筛查逐题(index 存储、语言无关)、is_novice 及其五个操作化子项、基线协变量 `aiexp/trust/own_trait`、语言、拉丁方 seq、状态漏斗(screened_out / in_progress / done 计数)、完成码。
- 注意力检查:`attention_ok`(仅对/错;原始作答没存——已知 nit)。
- 全程总时长:`participants.created_at`(=筛查提交)→ `final_survey_submit` 事件。
- 全程 token 成本:Σ `llm_done.usage.total_tokens`(7-02 起;此前的调用无此数据)。

### 轮级(trials 便捷列 + events 可复算)
- 条件 / 题目(topic_json 全文)/ 创意 / 终稿 / `parse_ok` / 模型参数(model/temperature/base_url 逐 trial)。
- **时间五列**(LLM 等待均已剔除或单列):
  - `t_read_intent`:读题+写创意(round_start→intent_submit)
  - `t_pregen`(仅 E):第 1 轮引导答题净时(guidance_shown→guidance_submit,扣窗内生成等待)
  - `t_postgen`:看稿+修改净时(首 script_shown→trial_submit,扣 AI 等待)
  - `t_llm_wait`:本轮全部 AI 等待(含失败重试)
  - `t_total`:round_start→trial_submit(不含问卷)
  - (问卷时长:events 算,见 §1④)
- **行为计数**:`n_ai_rounds`(D 修订轮 / E 追问轮)、`n_hand_edits`、`hand_edit_chars`。
- **E 引导剂量**(guidance_json):每问 自填率 is_custom / AI 代答率 ai_decided / 所选内容;问题数;fallback;追问轮数及每轮基于哪版稿(`draft_snapshot_ref`)。
- **D 反馈素材**:每条原文+顺序 → A5 编码、图4。
- **版本演化**:script_versions 全链 → 任意两版 diff、AI/人改比例、逐版长度轨迹、终稿↔首版 AI 稿距离(编辑距离/embedding,分析层)。
- **逐镜头**:parse_shots(终稿) × `shot_annotations_json`(mine/ai_ok/ai_against per shot)→ 逐镜头保真。
- **每轮问卷**:own1-3、soa1-2、tlx1、violation、imagine_match、satisfaction(全 1-7)。

### 事件级
- 每轮时间线完整重放(§0),毫秒级 ts + seq_in_round/attempt 排序分段(7-02 起;此前旧行秒级 ts、按 id 排序);每次 AI 调用的精确耗时、成败、token;续接/研究员操作留痕。
