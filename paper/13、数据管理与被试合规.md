## 1. 数据都有什么(敏感度分级)

| 数据 | 存哪 | 敏感度 | 备注 |
|---|---|---|---|
| 人口学/筛查(年龄段·专业·AI使用等) | `participants.demographics_json/screening_json` | 低-中 | **以 index 存储、语言无关**,非自由文本;最小化采集 |
| 基线协变量(aiexp/trust/own_trait) | 同上 | 低 | 刻度值 |
| **被试自由文本**(创意 intent / D 的反馈 / 手改稿 / final_survey 留言) | `trials.*` / `participants.final_survey_json` | **中-高** | 可能无意中含可识别信息(人名/学校/具体经历)——见 §4 |
| AI 生成的每版脚本 | `trials.script_versions` | 低 | |
| 量表作答 / 逐镜头标注 | `questionnaires.*` | 低 | |
| 行为事件流(时间戳/动作/token) | `events` | 低 | |
| 续接句柄 token | `participants.token` | 中 | 进网址 `?t=`,是会话恢复凭证,非身份 |
| 完成码 | 运行时生成 | 低 | 由 participant_id 派生 |

**不采集**:姓名、学号、邮箱、IP(Streamlit 默认不落库)、任何直接身份标识。participant_id = 自增整数,与真人身份之间**无映射表**(匿名而非假名)。

## 2. 存储与备份(自有服务器)

- **存储**:SQLite 单文件 `data/novastory.db`(WAL 模式),在你自己的服务器上原地持久,不用外部 DB。
- **备份(必须补——现仓库零备份)**:
  - 每日一次 + 建议每场被试完成后一次,用 SQLite 在线备份(WAL 安全):`sqlite3 data/novastory.db ".backup 'backup/novastory-YYYYMMDD-HHMM.db'"`;
  - 备份目录与主库**分盘/异地**至少一处(哪怕 rsync 到另一台/网盘);
  - 保留滚动 N 份 + 采数结束的最终快照。
- **`data/` 已 gitignore**(`.gitignore` 223-233 行),被试数据永不进 git。旧 `data/experiment_results.csv`(v1)与含 37 条开发数据的当前库,正式采数前归档到 `data/archive/`,从**空库起跑**。

## 3. 传输与访问控制

- **HTTPS**:被试要填人口学与自由文本,公开链接必须走 TLS(明文 HTTP 不可)。反代(Caddy/nginx)配证书。
- **研究员区**:`views/sidebar.py` 的研究员面板可浏览/导出全部数据、切条件、看实验目的。**部署时必须在 secrets 设强 `researcher_password`**——否则回退默认弱口令 `nova`,任何人可入 = 数据泄露 + 目的暴露([config.py:35](../core/config.py#L35))。
- **端口**:只暴露反代端口,防火墙挡直连 8501。

## 4. 自由文本的可识别信息风险

被试的创意/反馈/留言是自由输入,**可能无意中写入真实人名、校名、具体经历**。处置:
- 同意书说明"请勿填写可识别他人或自己隐私的真实信息";
- 分析/发表前对自由文本做一遍**人工去标识扫查**(尤其要进论文定性框的摘录);
- 定性摘录发表前改写/打码。

## 5. ⚠️ 第三方数据流出:被试文本会发送到 OpenAI

**这是最容易被忽略的合规点。** 被试写的创意、手改稿、D 的反馈,都会作为 prompt **发送到 OpenAI API**([core/llm.py](../core/llm.py))进行生成/引导。即:
- 被试的原创文本会**离开你的服务器、经过第三方(OpenAI)**;
- **同意书必须明确告知**"你输入的内容将由外部 AI 服务(OpenAI)处理以生成脚本";
- 参考 OpenAI API 数据政策(API 数据默认不用于训练、有限期留存),写进同意书/数据管理说明;
- 若研究科对数据出境敏感,这是要提前说清的点(采数用 OpenAI 是 B9 拍板,质量/延迟考量)。

## 6. 数据用途与保留

- **仅用于本研究**(学位论文 + 投稿 + 预注册的公开数据集若做则匿名后)。
- 不完整数据(未走到 trial_submit / final_survey 的 in_progress 被试)**弃用不进分析**(7-02 拍板①),但匿名记录可留档。
- 保留期:按学位/投稿要求(通常成果发表后数年),到期后处置。
