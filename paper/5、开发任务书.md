## 2. 目标架构(与 paper/7 §2 一一对应)

```
轮内状态机(state.r_phase):
  intent ──提交创意──▶ ┌ C: 直接生成 ──────────────▶ postgen
                       ├ D: 直接生成 ──────────────▶ postgen
                       └ E: guidance(第1轮问答) ──▶ 生成 ──▶ postgen
  postgen(看稿循环,D/E;C 仅展示+提交):
      渲染当前稿 v_n + 三按钮
      [提交] ──(先快照手改)──▶ questionnaire
      [告诉AI怎么改](仅D)→ 文字框 →(快照)→ AI修订 → v_{n+1}(ai) → postgen
      [让AI继续引导](仅E)→ 1-3问 → 作答 →(快照)→ AI修订 → v_{n+1}(ai) → postgen
      [自己改稿](D/E)→ 可编辑文本框 → [保存修改] → v_{n+1}(user_edit) → postgen
  questionnaire ──▶ 下一轮 / done        (问卷本身不变)
```

**快照硬规则的实现位**:三按钮的 on_click 第一步统一调 `_persist_pending_edit()`——若手改文本框开着且内容≠当前版,先落一版 `author=user_edit` 再执行动作。

**引导步模型策略**(C2 结论):质量 8/8 通过、延迟 ~75s。实现为 `config.GUIDANCE_API_INDEX: int|None`——None=跟随主配置;正式实验前重测延迟,超标则指到 OpenAI 配置(secrets 第 4 项)。等待期间 UI 显示进度文案("AI 正在为你准备问题,约 1 分钟…")。
> 注(2026-07-02):B9 拍板(正式全程 OpenAI)后本策略已简化——引导步跟随主配置即可,`GUIDANCE_API_INDEX` 保持 None。等待 UI 已于 7-02 深夜改为进度条 + 简短文案(AUD10,「约 1 分钟」措辞已删)。

## 3. 数据模型增量(迁移,不建新表)

> 注(2026-07-02):v2 旧列(ai_outline/edited_outline/dissent_json/adjudication*/t_edit/t_dissent)已从 `_SCHEMA` 移除(旧库中既有列保留无害);实际实现中 `t_pregen` 的起点是 guidance_shown(非 intent_submit),`hand_edit_chars` 用 difflib opcodes(非长度差)——以代码为准。
> 注(2026-07-02 深夜,LOG 批次):events 另加三列迁移 `seq_in_round`/`attempt`/`trial_id`(回填);`ts` 写入升毫秒(格式变更,非迁移);详见 paper/12 §0。

```sql
-- core/db.py _MIGRATIONS 追加:
ALTER TABLE trials ADD COLUMN guidance_json TEXT;       -- paper/7 §6 rounds 结构(E)
ALTER TABLE trials ADD COLUMN revision_requests TEXT;   -- [{round,text}](D)
ALTER TABLE trials ADD COLUMN script_versions TEXT;     -- [{v,author:"ai"|"user_edit",text}]
ALTER TABLE trials ADD COLUMN n_ai_rounds INTEGER;      -- D修订轮数 / E追加引导轮数
ALTER TABLE trials ADD COLUMN n_hand_edits INTEGER;
ALTER TABLE trials ADD COLUMN hand_edit_chars INTEGER;  -- 逐版长度差绝对值之和(精确diff留给分析层)
ALTER TABLE trials ADD COLUMN t_pregen REAL;            -- E:答题净时长(去LLM等待);C/D=NULL
ALTER TABLE trials ADD COLUMN t_postgen REAL;           -- 首版稿展示→提交,净时长
```

**events 新类型**:`guidance_shown(round,n_questions)`、`guidance_answer(dimension,is_custom,ai_decided)`、`guidance_submit(round)`、`revision_request(chars)`、`hand_edit_saved(v,chars_delta)`、`script_shown(v)`、`continue_guidance_click`;7-02 深夜增 `guidance_answer_saved(round,q)`(E 逐题作答时刻)。
**guidance_json 字段**以 paper/7 §6 为准(items 含 question/options/chosen/is_custom/ai_decided/fallback;追加轮含 draft_snapshot_ref)。

## 4. 任务分解(总估时 7-9 个工作日,兼职 ≈1.5-2 周)

### N2 Prompts 与 JSON 客户端(1 d)

  - `build_revision(topic, intent, current_script, request_text)`(D)与 `build_script_from_answers(topic, intent, answers)`(E 首稿)、`build_script_revision_from_answers(...)`(E 追加轮修订)——输出仍是标准分镜模板(shots.py 能解析)。
  验收:每个 prompt 用免费 key 手测 2 例,JSON 可解析/分镜可解析;修订确实保留未被点名的部分。
- **T2.2 `core/llm.py` 增 `generate_json(system,user,*,group,retries)`**:调用→`clean_output`→剥代码围栏→`json.loads`→失败重试(重试时附错误提示);全失败抛 `LLMJsonError`;支持 `GUIDANCE_API_INDEX` 覆盖客户端配置。
  验收:对故意注入坏返回的 mock 走通重试与异常路径。

### N3 E 条件:引导问答(2 d,核心)
- **T3.1 `views/guidance.py`(新,≈180 行)**:问答渲染组件——
  逐条展示("和 AI 一起完善你的故事 · 第 i/N 问"),每问=问题+`why`一行小字+选项(segmented/pills)+「✍ 自己写」输入框+「🤖 AI 帮我决定」;上一问/下一问导航;全部作答(或 AI 代答)后[提交回答]。答案存 session,提交时组装 guidance_json round 条目。
  **降级**:`LLMJsonError` 时该轮退化为一个开放问题("用几句话补充你想要的方向"),记 `fallback=true`。
  验收:AppTest 能以 session 注入完成作答;ai_decided/is_custom 正确落库。
- **T3.2 `views/group_e.py` 重写**:intent→调 round1 引导(等待文案)→T3.1 作答→`build_script_from_answers` 生成首稿(author=ai, v1)→进 postgen(N4);[让 AI 继续引导]→followup 1-3 问→作答→修订稿 v++。轮数无上限,`n_ai_rounds` 累加。
  验收:E2E 走"答题(含1个自己写+1个AI帮我决定)→首稿→继续引导一轮→提交",guidance_json 两轮结构完整。
  依赖:N2、T3.1、N4。

### N4 共用看稿循环 + D 重写(1.5 d)
- **T4.1 `views/_postgen.py`(新,≈150 行)**:看稿循环外壳(按 D24 修订版)——**脚本=常驻可编辑文本框**(预填当前版)+ 条件专属 AI 通道常驻(D=聊天输入框+发送,E=继续引导按钮)+ [提交];无手动保存:`persist_pending_edit()` 在提交/AI 动作时自动对比文本框与当前版,有差异先落 `author=user_edit` 快照(记 `hand_edit_saved`,字符差用 difflib 计)。
  验收:手改后直接点提交,提交版=手改版且产生 user_edit 快照;手改后点 AI 通道,AI 收到的是手改版。
- **T4.2 `views/group_d.py` 重写**:intent→直接生成首稿(同 C 的 script prompt)→postgen;[告诉 AI 怎么改]→文字框(placeholder 示例"更滑稽 / 删掉第2镜的台词 / 结局反转",页面文案明示三种玩法)→`build_revision` 修订→v++;`revision_requests` 逐条记录。
  验收:抽象反馈("更炸裂")能产生明显修订;循环 3 轮后 script_versions 含 4 版且 author 标注正确。
- **T4.3 `views/_trial.py`/`core/state.py`**:submit_trial 写新字段(guidance_json/revision_requests/script_versions/n_*/t_pregen/t_postgen);state 增 postgen 子态与新计时(round_durations 重写:t_pregen=intent_submit→guidance_submit 净时,t_postgen=首次 script_shown→trial_submit 净时);删 outline/dissent 相关 payload 键。
