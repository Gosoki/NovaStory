## 1. 总体流程(每个被试一次会话走完,约 10-15 分钟)

```
打开链接
  → 知情同意页(勾选 + 开始)
  → 背景问卷(9 道单选,约 1 分钟;不淘汰任何人,novice 只作记录)
  → 自动分配序列 seq(见 §2)
  → 第 1 轮 → 第 2 轮 → 第 3 轮
  → 完成页(显示 8 位完成码)
```

每一轮(task)的内部结构,所有条件相同:

```
读题卡(题目+情境+3镜头/15秒规格)
  → 写 1-2 句故事创意(≥10 字)
  → 条件流水线(C / D / E,见 §3)
  → 本轮问卷 + 逐镜头标注(见 §4)
```

页面顶部有进度条(第 i/3 轮)和步骤条(当前步高亮,如
`✓ 写创意 → ▶ ② 修改大纲 → ③ 听取 AI 意见 → ④ 生成脚本 → ⑤ 回答问题`)。

## 2. 被试分配:共 9 种情况(3×3 拉丁方)

**不是随机**:第 N 个完成背景问卷的被试拿 `seq = N % 9` 号序列(确定性轮转,保证小样本完美配平)。题目固定为 `data/topics.json` 前三个:**T1 期末周渡劫 / T2 错过的末班车 / T3 五分钟的告白**。

| seq | 第 1 轮 | 第 2 轮 | 第 3 轮 |
|-----|---------|---------|---------|
| 0 | C × T1 | D × T2 | E × T3 |
| 1 | C × T2 | D × T3 | E × T1 |
| 2 | C × T3 | D × T1 | E × T2 |
| 3 | D × T1 | E × T2 | C × T3 |
| 4 | D × T2 | E × T3 | C × T1 |
| 5 | D × T3 | E × T1 | C × T2 |
| 6 | E × T1 | C × T2 | D × T3 |
| 7 | E × T2 | C × T3 | D × T1 |
| 8 | E × T3 | C × T1 | D × T2 |

配平性质(已由自动测试验证):每个条件在每个轮次位置恰好出现 3 次;每个条件与每个题目的配对恰好出现 3 次。N=27 时每序列 3 人,N=30 时前 3 个序列各多 1 人。

另外每个被试按 `seq % 4` 固定分到一个 ModeMirror 发散维度(视角 / 时间结构 / 基调 / 类型混搭),保证 E 条件的"反提案"不会把所有人推向同一个方向。

## 3. 三种条件流水线

### C · 全自动(1 个 LLM 等待窗口)

```
写创意 → [自动] AI 直接生成 3 镜头终稿 → 确认 → 问卷
```
被试唯一的创作输入就是那 1-2 句创意。

### D · 大纲检查点(2 个等待窗口)

```
写创意 → [自动] AI 生成 3 句故事大纲
       → 被试在文本框中自由编辑大纲(可改写/删除/添加)
       → 点「确认大纲,生成最终脚本」→ AI 依大纲生成终稿 → 确认 → 问卷
```
记录 AI 原始大纲与编辑后大纲两份快照(编辑距离 = 干预剂量,离线计算)。

### E · 大纲检查点 + ModeMirror 异议(4 个等待窗口)

```
写创意 → [自动] AI 生成大纲 + 同时预采样 3 份「默认大纲」(一次 n=3 调用)
       → 被试编辑大纲
       → 点「完成编辑,听听 AI 的不同意见」
       → AI 输出三段异议:【重合】你与默认版本撞在哪 /【提问】挑战性问题 /【反提案】沿本被试的发散维度给替代构思
       → 强制三选一裁决(不选不能继续):
           ├─ 采纳        → 提示「请把反提案落实进大纲」→ 可再编辑
           ├─ 部分采纳    → 同上
           └─ 拒绝        → 提示可直接生成
         (均可附一句理由,选填)
       → 点「确认大纲,生成最终脚本」→ AI 依最终大纲生成终稿 → 确认 → 问卷
```
记录:3 份默认大纲、异议全文、发散维度、裁决选项、理由、裁决前后大纲快照。

**相邻条件只差一个要素**:C→D 加"人类编辑入口",D→E 加"AI 异议"。这是设计的核心(单变量递进,审稿不可拆解)。

## 4. 每轮问卷(全部为 1-7 分段按钮,强制作答)

| 块 | 题数 | 内容 |
|----|------|------|
| 心理所有权 | 3 | 是"我的"作品 / 归属感 / 反映我的想法和风格 |
| 主导感(agency) | 2 | 我在主导走向 / 结果在我掌控中 |
| 负荷 | 1 | 精神负荷大 |
| 意图违背 | 1 | 终稿多大程度违背本意 |
| 想象匹配 | 1 | 终稿和你想象中的画面有多接近 |
| 注意力检查 | 1(**仅第 2 轮**) | "请直接选 2" |
| 逐镜头标注 | 3(每镜头 1 题) | 主要是我的想法 / 主要是 AI 的但我认可 / 违背了我的本意 |

共 8-9 道 + 3 道镜头标注 ≈ 1 分钟/轮。

## 5. 分支与特殊情况

| 情况 | 行为 |
|------|------|
| 终稿解析失败(模型没按模板输出) | 逐镜头标注降级为整体 1 题;trial 记 `parse_ok=0` |
| LLM 调用失败 | 页面显示错误,点任意按钮自动重试;事件表记 `llm_error` |
| 创意 <10 字 / 大纲为空 / 问卷有漏答 | 拦截并提示,不能继续 |
| 被试中途关页 | participants 留 `in_progress`,不进主分析 |
| E 条件默认大纲采样失败 | 在请求异议时自动补采,再失败则停在编辑页可重试 |
| 注意力检查答错 | 正常完成实验,`attention_ok=0`,按预注册规则在分析层排除 |
| 非 novice 被试 | 正常完成实验,`is_novice=false`,进敏感性分析而非主分析 |

## 6. 时间预算(目标 10-15 分钟,上限 20)

| 环节 | 估时 |
|------|------|
| 同意 + 背景问卷 | ≈1.5 min |
| C 轮(1 次等待) | ≈2-2.5 min |
| D 轮(2 次等待 + 编辑) | ≈3.5-4 min |
| E 轮(4 次等待 + 编辑 + 裁决) | ≈4.5-5 min |
| 完成页 | ≈0.5 min |
| **合计** | **≈12-14 min** |

LLM 等待时间单独计入 `t_llm_wait`,不算进创作时间。

## 7. 数据落点(SQLite `data/novastory.db`,四张表)

| 表 | 何时写入 | 关键字段 |
|----|----------|----------|
| participants | 背景问卷提交时 | seq、人口学、screening(含 is_novice)、attention_ok、完成码、状态 |
| trials | 每轮终稿确认时 | 条件、题目快照、创意、AI/编辑后大纲、dissent_json、裁决+理由、终稿、parse_ok、模型/温度/base_url、分阶段时长 |
| events | 全程逐事件 | round_start / intent_submit / llm_start / llm_done / outline_shown / dissent_request / dissent_shown / adjudicate / final_click / final_shown / trial_submit / questionnaire_submit 等 |
| questionnaires | 每轮问卷提交时 | 所有权/agency/负荷 JSON、违背、想象匹配、逐镜头标注 JSON |

## 8. 实验员 / 测试路径(不影响被试)

侧边栏「实验员工具」(密码,默认 `nova`)解锁后:

- **实验员模式**:四表浏览 / 按被试过滤 / CSV 导出;
- **跳过同意+筛查**:注入带 `dev:true` 标记的测试被试(事后可过滤),直接进第 1 轮;
- **一键填充当前页输入**:创意 / 大纲编辑 / 裁决+理由 / 整页问卷自动填好;
- **切到 C/D/E**:随意切换当前轮条件(重置本轮,从写创意重来),并显示完整序列(如 `seq=0 | R1:C → R2:D → R3:E`)与当前阶段;
- **为下一位被试重置会话**(本地连续测试用)。

成套手动输入:`samples/test_inputs.md`;全自动回归:`.venv/bin/python scripts/dev_smoke_e2e.py`(LLM 打桩,不耗 API)。
