## 1. 总体流程(每个被试一次会话走完,约 10-15 分钟,软目标)

```
打开链接
  → 知情同意页(顶部选语言:日本語/中文,默认 ja,仅此一次;之后只有管理员能切)
      (研究流程概述 + 「勿刷新」提示 + 勾选同意 + 开始)
  → 背景问卷(12 项:9 道单选 + 3 项 7 点刻度,约 1-2 分钟;不淘汰任何人,novice 只作记录 is_novice)
  → 自动分配序列 seq(见 §2),并把续接 token 写入网址(?t=…)
  → 第 1 轮 → 第 2 轮 → 第 3 轮
  → 总问卷 final_survey(4 题:最喜欢哪轮 / 以后用哪轮 / 总体满意度 7 点 / 自由留言)
  → 完成页(显示 8 位完成码)
```

背景问卷 12 项 = 人口学与使用频率等 9 道单选 + 3 项 7 点刻度(创作自评 self_rating、AI 信任 trust、所有权倾向 own_trait;其中 AI 创作经验 aiexp、trust、own_trait 为 6-26 新增的事前基线协变量,前测防任务污染)。

每一轮的骨架:**读题卡 → 写 1-2 句故事创意(≥10 字)→ 条件流水线(§3)→ 本轮问卷(§4)**。
页面顶部有进度条与步骤条(当前步高亮),三条件的步骤条各自如实展示自己的流程。

**刷新/断网不丢数据(AUD6)**:续接 token 在网址里,刷新或重连后从 DB 重建身份与序列,跳回第一个没答完问卷的轮次(该轮从头重做,`INSERT OR REPLACE` 防重复行),计时锚点重新落 `round_start`。被试若丢弃带 token 的网址、从原始邀请链接重进,才会留下一行 `in_progress` 旧记录。

## 2. 被试分配:9 种序列(3×3 拉丁方,确定性轮转)

第 N 个通过筛查的**真实**被试拿 `seq = N % 9`(研究员注入的 dev 测试被试不计入,分配在写锁内原子完成,并发不会撞号);3 种条件顺序(CDE/DEC/ECD)× 3 种题目轮换;每个条件在每个轮次位置、与每个题目的配对各恰好 3 次。

题目 = `data/topics.json` 的 3 个日本日常微情境(**7-18 重设计,基调正交,paper/14 §3**):**拾った切符(誰かの落とし物)/ 最後のひと口(分け合う・独り占め)/ はじめての街の、最初の一歩**。字段为 `{ja, zh}` 双语字典(zh 供研究员测试)+ `shot_count`(3)/`total_seconds`(15)。**开放度已经机器基线 norming 校准可比(gzip_cr 极差~1%)。**

## 3. 三种条件流水线(核心对仗:事前引导 vs 事后修改)

### C · 一次成型(零回路)
```
写创意 → [自动] AI 生成完整 3 镜头脚本 → 看稿(只读)→ [提交]
```
不能改、不能反馈。对应现实:一键生成工具。

### D · 先生成,再修改(常规 ChatGPT 工作流的代理)
```
写创意 → [自动] AI 生成完整脚本
       → 看稿页:脚本本身是【常驻可编辑文本框】+「告诉 AI」聊天框常驻
            ├ 直接在稿子上手改(无需保存按钮)
            ├ 给 AI 反馈(可抽象如「更搞笑」,可具体)→ AI 修订 → 新版本
            └ [满意了,提交这一版]
       循环无上限
```

### E · 先引导,再生成(本研究提出的工作流)
```
写创意
→ 【第 1 轮引导】AI 一次生成 5-7 问(固定 3 维:主角心理/关键转折/核心画面 + AI 补 2-4 维)
     逐条展示:问题 + 为什么问 + 3-4 个选项 + 「✍ 自己写」 + 「🤖 交给 AI 决定」
     可回看上一问;答完一并提交
→ [自动] AI 按创意+回答生成完整脚本
→ 看稿页:可编辑文本框常驻 + 「让 AI 继续引导」按钮
     ├ 直接手改
     ├ 继续引导 → AI 读当前稿(含手改)出 1-3 个新问题 → 作答 → AI 修订
     └ [满意了,提交这一版]
   循环无上限
```
E 没有自由文字反馈通道(想表达具体想法走「自己写」或手改)。
引导问题生成期间(首轮与「继续引导」追问同)显示**进度条**(60 秒封顶缓动)+ 简短等待文案;脚本生成本身是流式直出,无需进度条。

**快照硬规则**:提交或任何 AI 动作前,若编辑框内容与当前版本不同,先自动落一版 `author=user_edit`;AI 修订永远基于用户最新稿。每版脚本全文入库。

**LLM 输出语言**跟随被试语言(`prompts.build_*` 的 `lang` 参数,调用点传 `i18n.get_lang()`);被试端只开放 ja/zh(en 仅管理员可切,LLM prompt 不支持英文)。

## 4. 每轮问卷

问卷**开头**先把终稿渲染成分镜表(No. / 景别·カメラ / 画面 / 情节 / 台词;3 镜连续编号 + 第 4 行留空)供被试对照;「画面」列为占位「今後実装予定」(未来:每镜生成分镜草图)。解析失败时退回纯文本。

题目:所有权 3 题 + 主导感 2 题 + 负荷 1 题 + 意图违背 1 题 + 想象匹配 1 题 + **本轮满意度 1 题(6-26 新增)** + 逐镜头标注 3 题(1-7 分段按钮,强制作答);**第 2 轮**含注意力检查("请选 2")。

## 5. 分支与特殊情况

| 情况 | 行为 |
|------|------|
| 引导问题 JSON 解析失败(重试 2 次后) | 该轮退化为 1 个开放问题("用几句话补充你想要的方向"),记 `fallback=true`,流程不断 |
| E 追问引导生成失败 | 报错 + 重试按钮 + **「返回草稿」出口**(不会锁死在引导页) |
| 终稿没按编号模板 | 解析器改用「时长/秒数字段」兜底切分(6-27);兜底判定不可靠时诚实失败:逐镜头标注降级为整体 1 题,`parse_ok=0` |
| LLM 调用失败 | 报错,重试按钮;events 记 `llm_error`;失败等待计入 `t_llm_wait`,不污染创作净时长 |
| 创意 <10 字 / 引导某问没作答 / 修改要求为空 / 问卷漏答 | 拦截提示 |
| D/E 用户全程不用 AI 通道、只手改或直接提交 | 允许,本身就是行为数据 |
| E 用户全部「交给 AI 决定」 | 允许,引导剂量趋零,记录在案 |
| 刷新 / 断网重连 | URL token 续接(见 §1),不丢已完成轮次、不重复占 seq |
| 中途彻底流失(丢弃网址) | participants 留 `in_progress`,不进主分析 |

## 6. 数据落点(SQLite 四表)

| 表 | 要点 |
|----|------|
| participants | demographics_json / screening_json(含 is_novice、aiexp/trust/own_trait 基线协变量)、`token`(续接句柄)、`final_survey_json`(总问卷)、status(in_progress/screened_out/done) |
| trials | `guidance_json`(E 的逐轮问答全记录)、`revision_requests`(D 的反馈原文)、`script_versions`(每版全文+作者 ai/user_edit)、`n_ai_rounds`、`n_hand_edits`、`hand_edit_chars`、`t_pregen`(E 答题净时,扣除窗内生成等待)、`t_postgen`(看稿后净时)、parse_ok、模型参数;唯一索引 `(participant_id, round_idx)` + INSERT OR REPLACE 防重复;v2 旧列(outline/ModeMirror 系)已于 7-02 从 schema 移除(旧库中的既有列保留无害) |
| events | round_start / intent_submit / guidance_shown / **guidance_answer_saved(E 逐题作答时刻,7-02)** / guidance_answer / guidance_submit / revision_request / hand_edit_saved / script_shown / continue_guidance_click / llm_start / **llm_done(payload 含 `usage`={prompt, completion, total_tokens},LOG6;网关不返时记 null)** / llm_error / trial_submit / questionnaire_submit / final_survey_submit / session_resumed / dev_switch_condition。**7-02 起每行**:ts 为毫秒级,并带 `seq_in_round`(轮内序号)/`attempt`(会话段 id,重做轮切段)/`trial_id`(提交后回填中签段)——详见 paper/12 §0 |
| questionnaires | 每轮量表 + imagine_match + **satisfaction(6-26 新增)** + 逐镜头标注;唯一索引同 trials |

核心行为量:**事前投入**(E 答题时长、自填率、AI代答率)、**事后修改量**(AI 轮数 + 手改字符 + t_postgen)、两者之和=总投入(论文三口径并报,paper/9 §4)。

## 7. 实验员 / 测试路径

侧边栏「实验员工具」(密码,默认 `nova`):模型连通性测试(通/不通+延迟);数据后台四表浏览/导出;语言切换(含 en,仅管理员);**跳过同意+筛查**(dev 标记,**不占用真实被试的拉丁方计数**);**一键填充当前页输入**(创意/引导作答/D 反馈/手改/整页问卷含满意度);**切到 C/D/E**(就地换当前轮条件并重开本轮,计时锚点自动重记);为下一位被试重置(清续接 token)。

手动输入清单:`samples/test_inputs.md`;全自动回归:`.venv/bin/python scripts/dev_smoke_e2e.py`(LLM 打桩,不耗 API,含续接回归测试)。

## 8. 模型配置

开发期 = secrets 第 1 个配置(edgefn 免费网关,当前 KAT-Coder;mimo 已退役);**正式实验全程 OpenAI(B9 拍板,2026-06-26,30 人 <$5)**——把 secrets 首个配置换成 OpenAI 即可,引导步同主模型(`GUIDANCE_API_INDEX=None` 跟随主配置)。所有调用的 model/temperature/base_url 逐 trial 入库;每次 LLM 调用的 token 用量记入 events(LOG6)。
