# NovaStory v3 实验流程说明

> 更新 2026-06-13,与当前代码完全对应(以代码为准的运行时文档)。
> 设计依据:`paper/7 v4.1`(交互规格)+ `paper/5 v2.0`(工程);v2(ModeMirror 版)流程见 git 历史。

---

## 1. 总体流程(每个被试一次会话走完,约 10-15 分钟,软目标)

```
打开链接
  → 知情同意页(勾选 + 开始)
  → 背景问卷(9 道单选,约 1 分钟;不淘汰任何人,novice 只作记录 is_novice)
  → 自动分配序列 seq(见 §2)
  → 第 1 轮 → 第 2 轮 → 第 3 轮
  → 完成页(显示 8 位完成码)
```

每一轮的骨架:**读题卡 → 写 1-2 句故事创意(≥10 字)→ 条件流水线(§3)→ 本轮问卷(§4)**。
页面顶部有进度条与步骤条(当前步高亮),三条件的步骤条各自如实展示自己的流程。

---

## 2. 被试分配:9 种序列(3×3 拉丁方,确定性轮转)

与 v2 相同:第 N 个完成背景问卷的被试拿 `seq = N % 9`;3 种条件顺序(CDE/DEC/ECD)× 3 种题目轮换;每个条件在每个轮次位置、与每个题目的配对各恰好 3 次(自动测试验证)。题目 = `data/topics.json` 前三个(期末周渡劫 / 错过的末班车 / 五分钟的告白)。

---

## 3. 三种条件流水线(核心对仗:事前引导 vs 事后修改)

### C · 一次成型(零回路)
```
写创意 → [自动] AI 生成完整 3 镜头脚本 → 看稿(只读)→ [提交]
```
不能改、不能反馈。对应现实:一键生成工具。

### D · 先生成,再修改(常规 ChatGPT 工作流的代理)
```
写创意 → [自动] AI 生成完整脚本
       → 看稿页:脚本本身是【常驻可编辑文本框】+「告诉 AI 你想怎么改」聊天框常驻
            ├ 直接在稿子上手改(无需保存按钮)
            ├ 给 AI 反馈(可抽象如「更搞笑」,可具体)→ AI 修订 → 新版本
            └ [满意了,提交这一版]
       循环无上限
```

### E · 先引导,再生成(本研究提出的工作流)
```
写创意
→ 【第 1 轮引导】AI 一次生成 5-7 问(固定 3 维:主角心理/关键转折/核心画面 + AI 补 2-4 维)
     逐条展示:问题 + 为什么问 + 3-4 个选项 + 「✍ 自己写」 + 「🤖 交给 AI 决定」
     可回看上一问;答完一并提交
→ [自动] AI 按创意+回答生成完整脚本
→ 看稿页:可编辑文本框常驻 + 「让 AI 继续引导」按钮
     ├ 直接手改
     ├ 继续引导 → AI 读当前稿(含手改)出 1-3 个新问题 → 作答 → AI 修订
     └ [满意了,提交这一版]
   循环无上限
```
E 没有自由文字反馈通道(想表达具体想法走「自己写」或手改)。

**快照硬规则**:提交或任何 AI 动作前,若编辑框内容与当前版本不同,先自动落一版 `author=user_edit`;AI 修订永远基于用户最新稿。每版脚本全文入库。

---

## 4. 每轮问卷(与 v2 相同)

所有权 3 题 + 主导感 2 题 + 负荷 1 题 + 意图违背 1 题 + 想象匹配 1 题 + 逐镜头标注 3 题(1-7 分段按钮,强制作答);**第 2 轮**含注意力检查("请选 2")。

---

## 5. 分支与特殊情况

| 情况 | 行为 |
|------|------|
| 引导问题 JSON 解析失败(重试 2 次后) | 该轮退化为 1 个开放问题("用几句话补充你想要的方向"),记 `fallback=true`,流程不断 |
| 终稿解析失败(没按模板) | 逐镜头标注降级为整体 1 题;`parse_ok=0` |
| LLM 调用失败 | 报错,点任意按钮重试;events 记 `llm_error` |
| 创意 <10 字 / 引导某问没作答 / 修改要求为空 / 问卷漏答 | 拦截提示 |
| D/E 用户全程不用 AI 通道、只手改或直接提交 | 允许,本身就是行为数据 |
| E 用户全部「交给 AI 决定」 | 允许,引导剂量趋零,记录在案 |
| 中途关页 | participants 留 `in_progress`,不进主分析 |

## 6. 数据落点(SQLite 四表;v3 新增列)

| 表 | v3 要点 |
|----|---------|
| participants | 同 v2(is_novice 在 screening_json 内) |
| trials | 新增:`guidance_json`(E 的逐轮问答全记录)、`revision_requests`(D 的反馈原文)、`script_versions`(每版全文+作者 ai/user_edit)、`n_ai_rounds`、`n_hand_edits`、`hand_edit_chars`、`t_pregen`(E 答题净时)、`t_postgen`(看稿后净时);v2 的大纲/ModeMirror 列停用不删 |
| events | 新类型:guidance_shown / guidance_answer / guidance_submit / revision_request / hand_edit_saved / script_shown / continue_guidance_click |
| questionnaires | 同 v2(含 imagine_match) |

核心行为量:**事前投入**(E 答题时长、自填率、AI代答率)、**事后修改量**(AI 轮数 + 手改字符 + t_postgen)、两者之和=总投入(论文三口径并报,paper/9 §4)。

---

## 7. 实验员 / 测试路径

侧边栏「实验员工具」(密码,默认 `nova`):数据后台四表浏览/导出;**跳过同意+筛查**(dev 标记);**一键填充当前页输入**(创意/引导作答[自动混入 1 个自己写+1 个 AI代答,并跳到最后一问]/D 反馈/手改/整页问卷);**切到 C/D/E**(就地换当前轮条件并重开本轮);为下一位被试重置。

手动输入清单:`samples/test_inputs.md`;全自动回归:`.venv/bin/python scripts/dev_smoke_e2e.py`(LLM 打桩,不耗 API)。

## 8. 模型配置

被试用的生成模型 = secrets 第 1 个配置(当前 mimo-v2.5-pro);**引导提问步可独立配置**(`core/config.py: GUIDANCE_API_INDEX`,默认跟随主配置;正式收数前若延迟 >30s 指向 OpenAI 配置)。所有调用的 model/temperature/base_url 逐 trial 入库。
